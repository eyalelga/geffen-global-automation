> סיכום שיחת תכנון מלאה | עודכן: יוני 2026  
> מסמך זה מתעד את כל ההחלטות הארכיטקטוניות, החומרה הנדרשת, ותוכנית הפריסה.

---

## חזון המערכת

מערכת אוטומציה שמתעדת כל אינטראקציה עם לקוח — WhatsApp, מייל, טלפון, Zoom, ופגישה פרונטלית — ממירה אותה לתמליל, מחלצת תובנות בעזרת AI, ומעדכנת אוטומטית את תיק הלקוח ב-Fireberry. הכל מתבצע בתוך הרשת הפנימית. שום הקלטה לא יוצאת מחוץ לארגון.

---

## ארכיטקטורה — שכבות המערכת

### מה כבר קיים (לא נוגעים)
- **שיחות טלפון** — מוקלטות ונשמרות ב-Fireberry
- **WhatsApp** — מנוהל דרך WATI

### מה מוסיפים
- **Zoom** — webhook אוטומטי → הורדת הקלטה → QNAP
- **פגישות פרונטליות** — הקלטה רציפה בחדרים → עיבוד לפי טריגר

---

## טריגר פגישה — Dashboard נציגים

**הקובץ:** `meeting-dashboard.html` — דף ווב שרץ על QNAP, מוצמד כטאב ראשון בדפדפן של כל נציג.

**זרימה:**
1. הנציג לוחץ **"התחל ›"** ליד שם הלקוח בתחילת הפגישה
2. n8n מקבל webhook: שם לקוח, חדר, נציג, שעת התחלה
3. הנציג לוחץ **"סיים ✓"** כשהלקוח יוצא
4. n8n מקבל webhook עם משך הפגישה → חותך הקלטה → מפעיל עיבוד

**עלות לנציג:** 2 לחיצות לפגישה. ללא הקלדה. ללא הפרעה לאווירה.

## תרשים טופולוגיה

```
ערוצי קלט
  WhatsApp (WATI) ──────────────────────┐
  Email (Gmail API) ────────────────────┤
  Zoom (webhook) ───────────────────────┤──→ Cloudflare Tunnel ──→ QNAP LAN
  פגישה פרונטלית (Pi + מיק) ───────────┘

QNAP (רשת פנימית)
  ├── n8n            ← מנוע אוטומציה
  ├── WhisperX       ← תמלול קול לטקסט (מקומי)
  ├── Ollama+Qwen2.5 ← ניתוח AI (מקומי)
  ├── ChromaDB       ← זיכרון וקטורי לחיפוש
  ├── Open WebUI     ← ממשק שאילתות לנציגים
  ├── Metabase       ← דשבורד אנליטיקה
  └── Cloudflare Tunnel ← חיבור webhooks

שירותי ענן (API בלבד)
  ├── Fireberry      ← CRM, מקור האמת
  ├── WATI           ← WhatsApp Business API
  └── Monday.com     ← משימות פנימיות בלבד
```

## זרימת נתונים — סיכום

```
נציג לוחץ "התחל" בדשבורד
         ↓
n8n: קבלת webhook (לקוח + חדר + שעה)
         ↓
[פגישה מתנהלת — הקלטה רציפה ב-Pi]
         ↓
נציג לוחץ "סיים"
         ↓
n8n: ffmpeg חותך קטע מדויק מהקלטה
         ↓
WhisperX: תמלול + זיהוי דוברים
         ↓
Qwen2.5: סיכום + התחייבויות + סנטימנט + צעד הבא
         ↓
Fireberry: עדכון תיק לקוח
ChromaDB: שמירה לחיפוש עתידי
WATI:     הודעה ללקוח עם צעד הבא (אופציונלי)
```

## שאלות פתוחות לפני פריסה

1. **כמה נציגים** — קובע עלות Fireberry ומספר עמדות לדשבורד
2. **דגם QNAP** — לאחר הזמנה, לוודא שהתוספת RAM נכונה לדגם הספציפי
3. **Fireberry API** — וידוא endpoint מדויק לפגישות (activities vs. meetings)
4. **WhatsApp Templates** — אישור Meta לתגובות אוטומטיות (לוקח 1–3 ימים)
5. **הסכמת הקלטה** — הכנת טופס חתימה ללקוחות + שלטי "חדר זה מוקלט"
